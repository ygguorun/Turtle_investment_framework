> 执行 Phase 2 时参考这些示例的计算步骤和精度要求。

## 示例 1: DCF — 蓝筹价值型（以乳制品龙头为原型）

### 基础数据

| 项目 | 值 | 来源 |
|------|-----|------|
| FCF (近 3 年均值) | 8,500 百万元 | §5 |
| 营收 CAGR (5yr) | 8.2% | §3 |
| 总市值 | 150,000 百万元 | §1 |
| 总股本 | 6,300 百万股 | §1 |
| 当前股价 | 23.81 元 | §1 |
| 现金 | 25,000 百万元 | §4 |
| 有息负债 | 12,000 百万元 | §4 |
| Rf | 2.30% | §14 |
| 有效税率 | 22% | §3 |
| 财务费用 | 380 百万元 | §3 |

### WACC 计算

```
Beta = 0.8（大盘蓝筹，市值 > 1000 亿）
ERP = 6.0%（A股）
Ke = 2.30% + 0.8 × 6.0% = 7.10%

有息负债均值 = (12,000 + 11,500) / 2 = 11,750
Kd_pre_tax = 380 / 11,750 = 3.23%

E_weight = 150,000 / (150,000 + 12,000) = 92.59%
D_weight = 12,000 / (150,000 + 12,000) = 7.41%

WACC = 7.10% × 92.59% + 3.23% × (1 - 22%) × 7.41%
     = 6.578% + 0.187%
     = 6.76%
```

### FCF 预测

```
g_hist (FCF CAGR) = 9.5%
g_conservative = 9.5% × 0.8 = 7.6%
g_terminal = 3.0%
g_fade = (7.6% + 3.0%) / 2 = 5.3%
g_fade2 = (5.3% + 3.0%) / 2 = 4.15%

Year 1: 8,500 × 1.076 = 9,146    → PV = 9,146 / 1.0676 = 8,567
Year 2: 9,146 × 1.076 = 9,841    → PV = 9,841 / 1.0676² = 8,632
Year 3: 9,841 × 1.053 = 10,362   → PV = 10,362 / 1.0676³ = 8,509
Year 4: 10,362 × 1.0415 = 10,792 → PV = 10,792 / 1.0676⁴ = 8,301
Year 5: 10,792 × 1.030 = 11,116  → PV = 11,116 / 1.0676⁵ = 8,007

PV_FCF = 8,567 + 8,632 + 8,509 + 8,301 + 8,007 = 42,016
```

### 终值与估值

```
TV = 11,116 × 1.03 / (0.0676 - 0.03) = 11,449 / 0.0376 = 304,495
PV_TV = 304,495 / 1.0676⁵ = 219,344

Enterprise Value = 42,016 + 219,344 = 261,360
Equity Value = 261,360 + 25,000 - 12,000 = 274,360
Intrinsic per share = 274,360 / 6,300 = 43.55 元
```

### 敏感性矩阵

| WACC \ g | 2.5% | 3.0% | 3.5% |
|----------|------|------|------|
| 5.76% | 55.12 | 61.83 | 70.45 |
| 6.76% | 39.87 | **43.55** | 48.12 |
| 7.76% | 30.51 | 32.88 | 35.68 |

### 结论

```
内在价值: 43.55 元/股
当前股价: 23.81 元/股
安全边际: (43.55 - 23.81) / 23.81 = 82.9% → 显著低估
```

## 示例 2: DDM — 高分红蓝筹（以电力公司为原型）

### 基础数据

| 项目 | 值 |
|------|-----|
| DPS 序列 | 0.68, 0.71, 0.73, 0.78, 0.82 元 |
| EPS 序列 | 1.12, 1.18, 1.22, 1.30, 1.36 元 |
| Payout ratio | 60.7%, 60.2%, 59.8%, 60.0%, 60.3% |
| DPS CAGR | 4.8% |
| Payout std / avg | 0.3% / 60.2% = 0.5% → 极稳定 |
| Ke | 7.10% |
| 当前股价 | 28.50 元 |

### 模型选择

```
DPS_CAGR = 4.8% < 5% ✓
Payout CV = 0.5% < 20% ✓
→ 使用 Gordon 单阶段模型
```

### Gordon 模型计算

```
ROE_avg = 15.5%
g_sustainable = ROE × (1 - Payout) = 15.5% × (1 - 60.2%) = 6.17%
g = min(DPS_CAGR, g_sustainable) = min(4.8%, 6.17%) = 4.8%

V = DPS_latest × (1 + g) / (Ke - g)
  = 0.82 × 1.048 / (0.071 - 0.048)
  = 0.859 / 0.023
  = 37.35 元/股
```

### 敏感性

| Ke \ g | 4.3% | 4.8% | 5.3% |
|--------|------|------|------|
| 6.10% | 47.66 | 66.08 | 113.63 |
| 7.10% | 31.97 | **37.35** | 47.72 |
| 8.10% | 23.59 | 26.03 | 30.65 |

### 结论

```
内在价值: 37.35 元/股
当前股价: 28.50 元/股
安全边际: (37.35 - 28.50) / 28.50 = 31.1% → 低估
```

## 示例 3: PEG — 成长型（以动力电池龙头为原型）

### 基础数据

| 项目 | 值 |
|------|-----|
| PE (TTM) | 28.5 |
| EPS (TTM) | 8.92 元 |
| 归母净利润 3yr CAGR | 35.2% |
| 当前股价 | 254.22 元 |

### PEG 计算

```
PEG = PE / G = 28.5 / 35.2 = 0.81
→ PEG 在 0.5-1.0 区间 → 低估
```

### 反解合理股价

```
Fair_PE (PEG = 1) = G × 1.0 = 35.2
Fair_Price = 35.2 × 8.92 = 313.98 元
```

### 敏感性

| G 假设 | PEG | Fair_PE | Fair_Price |
|--------|-----|---------|-----------|
| 30.2% | 0.94 | 30.2 | 269.38 元 |
| 35.2% | 0.81 | 35.2 | **313.98 元** |
| 40.2% | 0.71 | 40.2 | 358.58 元 |

### 结论

```
内在价值: 313.98 元/股
当前股价: 254.22 元/股
安全边际: (313.98 - 254.22) / 254.22 = 23.5% → 低估
```

## 示例 4: 定性调整 — 海尔智家模式（收入质量修正）

### 场景

Python 计算结果显示海尔智家估值偏低（安全边际 > 20%），但定性报告揭示报表利润增速有水分。

### Python 默认值

| 假设 | Python 默认 |
|------|-----------|
| FCF增长率 | 4.58%（历史营收CAGR 5.72% × 0.8）|
| WACC | 7.80% |
| g_terminal | 3.0%（A股默认）|
| DCF 内在价值 | 32.15 元/股 |

### 定性报告信号（D1 利润质量分解）

```
报表净利润增速: +4.39%
核心经营利润增速: -0.14%

非核心利润来源:
  - 汇兑收益: +1,023.53 百万元（海外收入占比 50%+）
  - 研发费用削减: +644.18 百万元（研发强度下降）
  - 其他非经营性: +312 百万元

→ 剔除非核心项后，主业利润实际下滑
```

### 调整过程

```
1. FCF增长率:
   Python默认 = 4.58%（基于报表营收CAGR）
   D1信号: 核心经营利润增速 = -0.14%，远低于报表增速
   调整: 下调至 1.5%（介于核心增速和报表增速之间，给予部分信用）
   操作: 从DCF 5×5敏感性表选取 g=1.5%/WACC=7.80% 的坐标

2. g_terminal:
   Python默认 = 3.0%
   D2信号: moat_rating = "较强"，moat_sustainability = "中等可持续"
   调整: 下调至 2.5%
   操作: 从敏感性表选取 g_terminal=2.5% 列

3. 综合:
   从敏感性表 [WACC=7.80%, g_terminal=2.5%] 读取调整后 DCF 内在价值 = 26.80 元/股
   vs Python默认 32.15 → 调整幅度 -16.6%
```

### 结论

```
Python估值: 32.15 元/股 → 安全边际 +22%（低估）
定性调整后: 26.80 元/股 → 安全边际 +2%（合理）
判断从"低估"变为"合理"，定性调整避免了高估陷阱。
```

## 常见计算错误提醒

| # | 错误 | 正确做法 |
|---|------|---------|
| 1 | WACC 中 D 使用账面债务 | 有息负债 = 短期借款 + 长期借款 + 应付债券；不含应付账款等经营性负债 |
| 2 | g_terminal ≥ WACC → TV 为负或无穷 | g_terminal 必须 < WACC；若接近则强制 g = WACC - 2% |
| 3 | FCF 全部为负时强行 DCF | 应标注"DCF 不适用"，降低权重或切换 PS/EV 方法 |
| 4 | PE Band 包含亏损年 PE < 0 | 剔除 PE < 0 和 PE > 100 的异常年份 |
| 5 | DDM 的 g > Ke | Gordon 模型无解；必须切换两阶段模型或降低 g |
| 6 | PEG 中 G 使用单年增长率 | 应使用 3 年或 5 年 CAGR，避免单年波动误导 |
| 7 | EV 转 Equity 忘记加回现金 | Equity = EV + Cash - Debt（现金是股东的） |
| 8 | 港股/美股未统一币种 | 全程使用同一币种（百万港元或百万美元），不混用 |
| 9 | 忽略少数股东权益 | DCF 的 Equity Value 已是全体股东的；若要归母，需扣除少数股东 |
| 10 | 敏感性矩阵只展示不解读 | 必须标出"关键转折点"：哪个参数变化会改变估值判断 |
